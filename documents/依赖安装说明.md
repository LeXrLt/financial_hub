# 爬虫 SKILL_SETUP 编写指南

本文档面向爬虫开发人员，说明如何为自己的爬虫编写 `SKILL_SETUP.md` 文件，以便系统安装 skill 能自动完成依赖安装和环境配置。

---

## 1. 什么是 SKILL_SETUP.md

`SKILL_SETUP.md` 是放置在每个爬虫根目录下的安装引导文件，供 AI Agent 或系统安装 skill 读取并自动执行。它描述了该爬虫从零到可运行所需的全部步骤。

系统根目录的 `SKILL_SETUP.md` 会遍历 `crawlers/` 下的每个子文件夹，找到并执行各自的 `SKILL_SETUP.md`。

---

## 2. 文件格式

采用 YAML frontmatter + Markdown 步骤的格式：

```markdown
---
name: {爬虫名称}-setup
description: 简短描述该爬虫的安装配置用途
metadata:
  openclaw:
    requires:
      bins: ["python3"]  # 列出所需的系统命令
---

# {爬虫名称} 环境配置 Skill

简要说明本 skill 的用途。

## When to use

描述什么情况下应该执行这个 skill。

## Step 1: ...

具体安装步骤。

## Step 2: ...

...
```

## 3. 必须包含的步骤

### 3.1 创建虚拟环境（Python 项目）

```markdown
## Step N: Create virtual environment

Check if `{baseDir}/.venv` exists. If not, create it:

\```bash
python3 -m venv {baseDir}/.venv
\```
```

### 3.2 安装依赖

```markdown
## Step N: Install dependencies

\```bash
{baseDir}/.venv/bin/pip install -r {baseDir}/requirements.txt
\```
```

### 3.3 配置环境变量

```markdown
## Step N: Configure environment variables

Check if `{baseDir}/.env` exists. If not, copy from example:

\```bash
cp {baseDir}/.env.example {baseDir}/.env
\```

Then fill in the database configuration values.
```

### 3.4 验证安装

```markdown
## Step N: Verify setup

\```bash
{baseDir}/.venv/bin/python {baseDir}/some_verify_command.py
\```
```

## 4. 数据库环境变量统一规范

**所有爬虫必须使用以下统一的数据库环境变量名称和格式：**

### 4.1 必填变量

| Variable | Description | Default |
|---|---|---|
| `POSTGRES_HOST` | PostgreSQL 服务器地址 | `127.0.0.1` |
| `POSTGRES_PORT` | PostgreSQL 服务器端口 | `5432` |
| `POSTGRES_USER` | 数据库用户（读写权限） | `hub_user` |
| `POSTGRES_PASSWORD` | 数据库用户密码 | `hub_password` |
| `POSTGRES_DB` | 数据库名称 | `financial_hub` |

### 4.2 可选变量（如果爬虫有只读查询功能）

| Variable | Description | Default |
|---|---|---|
| `POSTGRES_READONLY_USER` | 只读用户 | `hub_readonly` |
| `POSTGRES_READONLY_PASSWORD` | 只读用户密码 | `hub_password` |

### 4.3 .env.example 模板

每个爬虫必须提供 `.env.example` 文件，内容如下：

```env
# Database (统一配置，请勿修改变量名)
POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=5432
POSTGRES_USER=hub_user
POSTGRES_PASSWORD=hub_password
POSTGRES_DB=financial_hub

# 以下为可选，如果爬虫有只读查询功能
# POSTGRES_READONLY_USER=hub_readonly
# POSTGRES_READONLY_PASSWORD=hub_password
```

### 4.4 为什么要统一

- 系统安装 skill 会自动将根目录 `.env` 中的数据库配置分发给每个爬虫
- 统一变量名确保自动化脚本能正确填写
- 所有爬虫连接同一个 PostgreSQL 实例的同一个数据库 `financial_hub`
- 每个爬虫操作自己来源的表，互不干扰

### 4.5 在代码中读取

Python 示例：

```python
import os
from dotenv import load_dotenv

load_dotenv()

DB_CONFIG = {
    "host": os.getenv("POSTGRES_HOST", "127.0.0.1"),
    "port": int(os.getenv("POSTGRES_PORT", "5432")),
    "user": os.getenv("POSTGRES_USER"),
    "password": os.getenv("POSTGRES_PASSWORD"),
    "dbname": os.getenv("POSTGRES_DB"),
}
```

如果项目中使用了 `financial-hub-postgres` 共享库，直接导入即可，库内部会自动读取这些变量。

## 5. 完整 SKILL_SETUP.md 示例

以下是一个 Python 爬虫的完整示例，可直接复制修改：

```markdown
---
name: my-crawler-setup
description: Install dependencies and configure environment for the My Crawler project.
metadata:
  openclaw:
    requires:
      bins: ["python3"]
---

# My Crawler 环境配置 Skill

本 skill 完成 My Crawler 的依赖安装和环境变量配置。

**本 skill 不涉及数据操作，仅做环境初始化。**

## When to use

Use this skill when:
- The project is freshly cloned and has not been set up yet
- The virtual environment `.venv/` does not exist
- The `.env` file does not exist
- The user explicitly asks to install or set up the project
- Other skills fail due to missing dependencies or missing `.env`

## Step 1: Create virtual environment

Check if `{baseDir}/.venv` exists. If not, create it:

\```bash
python3 -m venv {baseDir}/.venv
\```

## Step 2: Install Python dependencies

\```bash
{baseDir}/.venv/bin/pip install -r {baseDir}/requirements.txt
\```

Key dependencies (defined in `requirements.txt`):
- `requests` — HTTP client
- `psycopg2-binary` — PostgreSQL driver
- `python-dotenv` — Load `.env` config

## Step 3: Configure environment variables

Check if `{baseDir}/.env` exists. If not, copy from example:

\```bash
cp {baseDir}/.env.example {baseDir}/.env
\```

Then ask the user to confirm the database configuration values:

| Variable | Description | Default |
|---|---|---|
| `POSTGRES_HOST` | PostgreSQL server address | `127.0.0.1` |
| `POSTGRES_PORT` | PostgreSQL server port | `5432` |
| `POSTGRES_USER` | Database user (read-write) | `hub_user` |
| `POSTGRES_PASSWORD` | Password | `hub_password` |
| `POSTGRES_DB` | Database name | `financial_hub` |

**Important:** The `.env` file contains sensitive credentials and is already in `.gitignore`. Never commit it.

## Step 4: Verify setup

Run a simple test to verify the database connection:

\```bash
{baseDir}/.venv/bin/python -c "
import psycopg2, os
from dotenv import load_dotenv
load_dotenv('{baseDir}/.env')
conn = psycopg2.connect(
    host=os.getenv('POSTGRES_HOST', '127.0.0.1'),
    port=int(os.getenv('POSTGRES_PORT', '5432')),
    user=os.getenv('POSTGRES_USER'),
    password=os.getenv('POSTGRES_PASSWORD'),
    dbname=os.getenv('POSTGRES_DB'),
)
print('Database connection successful')
conn.close()
"
\```

If this command prints 'Database connection successful', the setup is complete.
```

---

## 6. 注意事项

1. **变量名必须统一** — 不要自创变量名如 `DB_HOST`、`PG_USER` 等，必须使用上述规范名称
2. **不要提交 `.env`** — 确保 `.gitignore` 中包含 `.env` 和 `.venv/`
3. **必须提供 `.env.example`** — 作为配置模板供安装 skill 和用户参考
4. **`{baseDir}` 占位符** — SKILL_SETUP.md 中使用 `{baseDir}` 代表该爬虫项目的根目录绝对路径，由执行 skill 的 agent 自动替换
5. **schema.sql** — 如果爬虫需要创建自己的数据表，在项目中提供 `schema.sql`，并在 SKILL_SETUP.md 中添加建表步骤
6. **验证步骤不可省略** — 必须有一个验证数据库连接的步骤，确保安装后即可运行

## 7. 目录结构参考

一个标准的爬虫项目应包含：

```
crawlers/{name}/
├── .env.example        ← 环境变量模板（必须）
├── .env                ← 实际配置（不提交 git）
├── .venv/              ← Python 虚拟环境（不提交 git）
├── .gitignore          ← 忽略 .env 和 .venv（必须）
├── requirements.txt    ← Python 依赖列表（必须）
├── schema.sql          ← 数据库表定义（如需建表）
├── main.py             ← 爬虫主入口
├── SKILL_SETUP.md      ← 安装配置 skill（必须）
├── SKILL.md            ← 功能 skill（可选）
└── README.md           ← 用户文档
```
