# Hub 系统数据库表说明

本文档描述 Hub 监控系统使用的所有数据库表，包括每张表的功能、字段含义、示例值和枚举可选值。

---

## 1. crawl_targets — 抓取目标表

### 功能

存储所有数据来源的抓取目标，Hub 通过此表管理和监控各爬虫的抓取对象。每个目标代表一个需要定时抓取的数据源（如某个公众号、某个 YouTube 频道、某个播客节目）。

### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例值 |
|------|------|------|--------|------|--------|
| id | SERIAL | 自增主键 | — | 唯一标识 | `1` |
| source_type | VARCHAR(50) | 是 | — | 数据来源类型 | `wechat` |
| target_name | VARCHAR(255) | 是 | — | 目标名称，人类可读 | `半佛仙人` |
| target_identifier | VARCHAR(500) | 是 | — | 目标标识，用于爬虫定位 | `gh_xxxxxxxxxxxx` |
| enabled | BOOLEAN | 否 | `true` | 是否启用抓取 | `true` |
| cron_expression | VARCHAR(100) | 否 | `0 */6 * * *` | 抓取频率，cron 表达式 | `0 */6 * * *` |
| last_crawl_at | TIMESTAMPTZ | 否 | NULL | 最后一次抓取时间 | `2024-03-15T10:30:00+08:00` |
| last_crawl_status | VARCHAR(50) | 否 | NULL | 最后一次抓取状态 | `success` |
| last_error | TEXT | 否 | NULL | 最后一次失败的错误信息 | `timeout after 30s` |
| total_items | INTEGER | 否 | `0` | 累计抓取到的数据条数 | `128` |
| notes | TEXT | 否 | NULL | 备注信息 | `每日更新，重点关注` |
| created_at | TIMESTAMPTZ | 否 | NOW() | 记录创建时间 | `2024-01-01T00:00:00+08:00` |
| updated_at | TIMESTAMPTZ | 否 | NOW() | 记录更新时间 | `2024-03-15T10:30:00+08:00` |

### 枚举值

**source_type** — 数据来源类型：

| 值 | 说明 |
|----|------|
| `wechat` | 微信公众号 |
| `youtube` | YouTube 频道/播放列表 |
| `xiaoyuzhou` | 小宇宙播客 |
| `cninfo` | 巨潮资讯网（A股公告） |

**last_crawl_status** — 最后抓取状态：

| 值 | 说明 |
|----|------|
| `pending` | 等待执行 |
| `running` | 正在执行 |
| `success` | 执行成功 |
| `failed` | 执行失败 |

### 索引

- `idx_crawl_targets_source` — 按 source_type 查询
- `idx_crawl_targets_enabled` — 按启用状态筛选

---

## 2. crawl_runs — 抓取运行日志表

### 功能

记录每次爬虫运行的详细信息，包括开始时间、结束时间、抓取结果和错误。用于追踪历史运行记录和排查问题。

### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例值 |
|------|------|------|--------|------|--------|
| id | SERIAL | 自增主键 | — | 唯一标识 | `42` |
| target_id | INTEGER | 是 | — | 关联的抓取目标 ID（外键） | `1` |
| status | VARCHAR(50) | 是 | `running` | 本次运行状态 | `success` |
| started_at | TIMESTAMPTZ | 否 | NOW() | 运行开始时间 | `2024-03-15T10:00:00+08:00` |
| finished_at | TIMESTAMPTZ | 否 | NULL | 运行结束时间 | `2024-03-15T10:05:12+08:00` |
| items_found | INTEGER | 否 | `0` | 本次发现的数据条数 | `15` |
| items_new | INTEGER | 否 | `0` | 本次新增的数据条数 | `3` |
| items_failed | INTEGER | 否 | `0` | 本次处理失败的条数 | `1` |
| error_message | TEXT | 否 | NULL | 错误信息（失败时填写） | `Connection refused to API` |
| duration_ms | INTEGER | 否 | NULL | 运行耗时（毫秒） | `5200` |

### 枚举值

**status** — 运行状态：

| 值 | 说明 |
|----|------|
| `running` | 正在运行 |
| `success` | 运行成功 |
| `failed` | 运行失败 |

### 外键

- `target_id` → `crawl_targets(id)`，级联删除

### 索引

- `idx_crawl_runs_target` — 按目标 ID 查询
- `idx_crawl_runs_started` — 按开始时间倒序排列

---

## 3. component_status — 组件健康状态表

### 功能

记录各系统组件（爬虫、转录器等）的健康状态。组件定期主动上报状态，Hub 据此判断各组件是否正常运行。使用 UPSERT 逻辑，每个组件名称唯一。

### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例值 |
|------|------|------|--------|------|--------|
| id | SERIAL | 自增主键 | — | 唯一标识 | `1` |
| component_name | VARCHAR(100) | 是（UNIQUE） | — | 组件名称 | `wechat_crawler` |
| status | VARCHAR(50) | 是 | `unknown` | 组件当前状态 | `healthy` |
| last_heartbeat | TIMESTAMPTZ | 否 | NULL | 最后一次心跳时间 | `2024-03-15T10:29:55+08:00` |
| last_error | TEXT | 否 | NULL | 最近一次错误信息 | `disk space low` |
| metadata | JSONB | 否 | `{}` | 附加元数据 | `{"version": "1.2.0", "pid": 1234}` |
| updated_at | TIMESTAMPTZ | 否 | NOW() | 记录更新时间 | `2024-03-15T10:29:55+08:00` |

### 枚举值

**component_name** — 组件名称（约定值）：

| 值 | 说明 |
|----|------|
| `wechat_crawler` | 微信公众号爬虫 |
| `youtube_crawler` | YouTube 爬虫 |
| `xiaoyuzhou_crawler` | 小宇宙播客爬虫 |
| `cninfo_crawler` | 巨潮资讯网爬虫 |
| `transcriber` | 音频转录服务 |

**status** — 组件状态：

| 值 | 说明 |
|----|------|
| `healthy` | 正常运行 |
| `degraded` | 运行中但性能下降或部分功能异常 |
| `down` | 已停止或无法响应 |
| `unknown` | 未知状态（未收到心跳） |

## 4. system_events — 系统事件日志表

### 功能

记录系统中发生的重要事件，用于审计和问题追踪。事件由各组件主动写入，Hub 可以按时间浏览。

### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例值 |
|------|------|------|--------|------|--------|
| id | SERIAL | 自增主键 | — | 唯一标识 | `100` |
| event_type | VARCHAR(100) | 是 | — | 事件类型 | `crawl_start` |
| source | VARCHAR(100) | 否 | NULL | 事件来源组件 | `wechat_crawler` |
| message | TEXT | 否 | NULL | 事件描述 | `开始抓取公众号「半佛仙人」` |
| metadata | JSONB | 否 | `{}` | 附加数据 | `{"target_id": 1, "trigger": "cron"}` |
| created_at | TIMESTAMPTZ | 否 | NOW() | 事件发生时间 | `2024-03-15T10:00:00+08:00` |

### 枚举值

**event_type** — 事件类型：

| 值 | 说明 |
|----|------|
| `crawl_start` | 爬虫开始执行 |
| `crawl_end` | 爬虫执行完毕 |
| `crawl_error` | 爬虫执行出错 |
| `target_enabled` | 抓取目标被启用 |
| `target_disabled` | 抓取目标被停用 |

### 索引

- `idx_system_events_created` — 按创建时间倒序排列

---

## 5. data_stats — 数据统计快照表

### 功能

定期记录各数据表的统计信息（行数、最新数据时间），用于在 Dashboard 展示数据增长趋势。可由定时任务定期写入快照。

### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 | 示例值 |
|------|------|------|--------|------|--------|
| id | SERIAL | 自增主键 | — | 唯一标识 | `50` |
| source_type | VARCHAR(50) | 是 | — | 数据来源类型 | `youtube` |
| table_name | VARCHAR(100) | 是 | — | 统计的表名 | `youtube_videos` |
| row_count | INTEGER | 否 | `0` | 表中的数据行数 | `256` |
| latest_item_at | TIMESTAMPTZ | 否 | NULL | 表中最新一条数据的时间 | `2024-03-15T08:00:00+08:00` |
| snapshot_at | TIMESTAMPTZ | 否 | NOW() | 快照时间 | `2024-03-15T10:00:00+08:00` |

### 枚举值

**source_type** — 数据来源类型（同 crawl_targets）：

| 值 | 说明 |
|----|------|
| `wechat` | 微信公众号 |
| `youtube` | YouTube |
| `xiaoyuzhou` | 小宇宙播客 |
| `cninfo` | 巨潮资讯网 |

### 索引

- `idx_data_stats_source` — 按 source_type + snapshot_at 倒序查询

## 表关系图

```
crawl_targets (1) ──< (N) crawl_runs
     │
     │  target_id FK
     │
component_status    (独立表，UPSERT by component_name)
system_events       (独立表，只追加)
data_stats          (独立表，快照)
```

## 连接池与性能说明

- Postgres `max_connections = 50`
- Hub 应用连接池 `max = 5`，空闲超时 10s 自动释放
- 为 Crawler、pgweb 等其他服务预留充足连接数
